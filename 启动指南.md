# 智慧经营 · 本地启动指南（Plan A 单机运行）

> 适用场景：在个人电脑（如 Mac Mini）独立运行前端 + 后端 + 数据库，**无需联网、无需云**。
> 技术栈：FastAPI 后端 + Vue 3 前端 + SQLite（默认）。前端使用 hash 路由，构建后由后端**同源托管**，所以日常只需起后端。

---

## 一、首次部署（只需做一次）

### 1. 拉取代码

```bash
git clone https://github.com/szlxjqr/mfr-cloud-finance.git
cd mfr-cloud-finance
```

> 若用 SSH（需先在 GitHub 配置 SSH 密钥）：`git clone git@github.com:szlxjqr/mfr-cloud-finance.git`

### 2. 前端：装依赖 + 构建

```bash
cd frontend
npm install          # 首次需装依赖（jszip / pdfjs-dist / tesseract.js 体积较大，稍慢）
npm run build        # 生成 frontend/dist/，后端会托管它
```

### 3. 后端：建虚拟环境 + 装依赖（推荐，不污染系统 Python）

```bash
cd backend
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

### 4. 启动并访问

```bash
# 后端（保持终端运行）
python -m uvicorn app.main:app --port 8521
```

浏览器打开：**http://localhost:8521**

> 数据库：首次启动会自动在 `backend/smart_finance.db` 创建 SQLite 文件，数据本地落盘。

---

## 二、日常启动（每次要用的时候）

**只需要起后端**，前端已经构建在 `dist/` 里、由后端同源托管：

```bash
# 终端 1：后端
cd <项目根目录>/backend
source .venv/bin/activate
python -m uvicorn app.main:app --port 8521
```

然后浏览器打开 **http://localhost:8521** 即可。

> 说明：只要没改前端代码、且 `frontend/dist/` 已存在，就**不需要**再跑 `npm run build`，也不需要单独起前端服务。

---

## 三、更新代码之后

```bash
cd <项目根目录>
git pull

# 若后端依赖 requirements.txt 有变动：
cd backend && source .venv/bin/activate && pip install -r requirements.txt

# 若前端有变动（或不确定）：
cd ../frontend && npm run build

# 重启后端：先 Ctrl+C 关掉旧进程，再重新起
cd ../backend && python -m uvicorn app.main:app --port 8521
```

## 四、常见问题

| 现象 | 处理 |
|------|------|
| `pip` 报 `externally-managed-environment` | 用上面的 **venv** 方案避开；不要 `--break-system-packages` 污染系统 |
| 端口被占用 | 改用 `--port 其他端口` 即可。Plan A 单机模式请求走相对路径 `/api`，**前端无需重建**；若用 dev 模式（`npm run dev`）联调，需把 `vite.config.ts` 里 proxy 的端口改成一致 |
| 页面空白 / 改了前端不生效 | 重跑 `npm run build` 再刷新浏览器 |
| 关掉后端终端后页面打不开 | 后端进程被关了，保持终端运行即可 |
| Python 版本兼容 | 代码已兼容 **Python 3.9+**（联合类型统一用 `Optional`）；建议 3.9 / 3.10 / 3.11 |

## 五、目录速查

| 路径 | 作用 |
|------|------|
| `backend/` | FastAPI 后端，用 `uvicorn` 启动 |
| `backend/smart_finance.db` | SQLite 数据库（首次启动自动生成，已 gitignore） |
| `frontend/` | Vue 3 前端源码 |
| `frontend/dist/` | 构建产物，由后端在 8521 端口托管 |
| `.venv/` | 后端虚拟环境（已 gitignore） |

---

## 六、换数据库（可选，未来上云再做）

默认用本地 SQLite，零配置。若以后要上云 Postgres，只需设置环境变量，无需改代码：

```bash
export DATABASE_URL="postgresql+psycopg://user:pass@host:5432/dbname"
```

（对应的 `psycopg` 依赖需在 `requirements.txt` 中补充后重新安装。）
