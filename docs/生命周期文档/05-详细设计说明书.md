# 详细设计说明书（LLD）

> 状态：核心模块（合同/税务/报销）已充实；其余沿用 04 概要设计 + 06 数据库设计。

## 1. 合同管理

### 1.1 实体
- `hr_contracts`（人事合同）：员工、合同类型、起止日、状态、薪资、附件。
- `sales_contracts`（销售合同）：合同号、客户、金额/税率/税额、状态、附件。
- `purchase_contracts`（采购合同）：结构同销售，对方为供应商。
- `parties`（往来单位）：客户/供应商档案，被销售/采购合同关联。

### 1.2 状态机
- 人事合同：`生效 → 到期 / 终止 / 续签中`
- 销售/采购：`草稿 → 执行中 → 已完成 / 终止 / 纠纷`

### 1.3 关键流程
合同签立 → **自动计提印花税**（写入 `tax_ledger`）→ 执行中跟踪收付款 → 完结。

## 2. 税务管理

### 2.1 核心实体
- `tax_ledger`（税款台账）：税种、所属期、应缴/已缴/未缴、申报状态、数据来源。

### 2.2 数据汇聚
- 印花税：`source=合同计提`（由合同签立触发）。
- 增值税：`source=发票汇总`（汇聚进/销项发票税额）。
- 个税：`source=工资计算`（链工资模块）。

### 2.3 工作台
按税种 + 所属期聚合展示应缴/已缴/未缴与申报状态，下钻看来源明细。

## 3. 报销管理

### 3.1 实体
- `reimbursement_requests`：单头（申请人/部门/状态/总额/附件）。
- `reimbursement_items`：明细（费用类型/金额/关联发票）。

### 3.2 状态机
`草稿 → 待审 → 已审 → 已付 / 驳回`

### 3.3 关键流程
提交 → 部门/财务审批 → 通过后生成**费用发票 + 出纳付款单**（与发票/出纳模块打通）。
