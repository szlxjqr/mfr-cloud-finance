# 数据库设计说明书

> 状态：已充实（新增模块；现有账务表待补实后端时补充）。

## 1. 实体关系概述
```
parties(客户/供应商) <-- sales_contracts / purchase_contracts
hr_contracts(独立，关联员工)
sales/purchase_contracts --签立--> tax_ledger(印花税)
发票(进/销项) --税额--> tax_ledger(增值税) / 发票税务汇总
工资 --个税--> tax_ledger(个税)
reimbursement_requests --审批通过--> 费用发票 + 出纳付款
```

## 2. 往来单位 `parties`

| 字段 | 类型 | 说明 |
|---|---|---|
| id | PK | |
| name | VARCHAR | 名称 |
| tax_no | VARCHAR | 税号 |
| type | VARCHAR | customer / supplier / employee_company |
| contact / phone / address | VARCHAR | 联系方式 |
| status | VARCHAR | 启用 / 停用 |

## 3. 人事合同 `hr_contracts`

| 字段 | 类型 | 说明 |
|---|---|---|
| id | PK | |
| employee_name | VARCHAR | 员工姓名 |
| id_number | VARCHAR | 身份证（脱敏 `*` 原样保留） |
| contract_type | VARCHAR | 劳动合同/劳务/实习/保密/竞业 |
| party_a / party_b | VARCHAR | 甲方(公司)/乙方(员工) |
| start_date / end_date | DATE | 起止 |
| status | VARCHAR | 生效/到期/终止/续签中 |
| salary | DECIMAL | 薪资（可选） |
| attachment_path | VARCHAR | 附件 |

## 4. 销售合同 `sales_contracts`

| 字段 | 类型 | 说明 |
|---|---|---|
| id | PK | |
| contract_no | VARCHAR | 合同号 |
| customer_id | FK→parties | 客户 |
| sign_date / effective_date / expire_date | DATE | 签立/生效/到期 |
| amount | DECIMAL | 含税金额 |
| tax_rate / tax_amount | DECIMAL | 税率/税额 |
| status | VARCHAR | 草稿/执行中/已完成/终止/纠纷 |
| attachment_path | VARCHAR | 附件 |

## 5. 采购合同 `purchase_contracts`
同销售合同，但 `supplier_id` FK→parties，对方为供应商。

## 6. 报销 `reimbursement_requests` / `reimbursement_items`

requests: `req_no, applicant, dept, submit_date, status(草稿/待审/已审/已付/驳回), total_amount, attachment_path`
items: `request_id(FK), expense_type, amount, invoice_id(可选), remark`

## 7. 税款台账 `tax_ledger`

| 字段 | 类型 | 说明 |
|---|---|---|
| id | PK | |
| tax_category | VARCHAR | 增值税/企业所得税/个税/印花税 |
| period | VARCHAR | 所属期（如 2026-06） |
| payable / paid / unpaid | DECIMAL | 应缴/已缴/未缴 |
| filing_status | VARCHAR | 未申报/已申报/已缴清 |
| source | VARCHAR | 手工/发票汇总/合同计提/工资计算 |
| related_no | VARCHAR | 关联单号 |

## 8. 通用约定
- 所有表含 `id`(PK)、`created_at`、`updated_at`、`remark`。
- 金额用 `DECIMAL(18,2)`。
