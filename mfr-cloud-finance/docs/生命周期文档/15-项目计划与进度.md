# 项目计划与进度

> 状态：已充实（阶段划分）；角色与排期为占位。

## 1. 阶段划分（Phase）

| 阶段 | 内容 | 关键产出 |
|---|---|---|
| Phase 0 地基 | 补 `app/db/` 层 + 抽前端共享 `http` 实例 + （可选）mock 登录 | 数据库可连、统一请求层 |
| Phase 1 合同管理 | parties + 三张合同表 + 列表/表单/模板 + 到期提醒 | 合同模块可用 |
| Phase 2 报销管理 | 申请/审批/台账 + 状态机 + 关联费用发票/出纳 | 报销闭环 |
| Phase 3 税务管理+打通 | tax_ledger + 工作台 + 合同印花税自动计提 + 发票税额汇总 | 税务汇聚视图 |
| Phase 4 工资/资产/账务补实 | 现有占位页按表结构补后端 CRUD | 全模块可用 |

## 2. 里程碑
- M1：地基完成，首个真实 CRUD 跑通。
- M2：合同管理上线。
- M3：报销 + 税务打通上线。
