# 概要设计说明书（HLD）

> 状态：已充实。

## 1. 设计原则
- **业财分离 + 场景化**：一级菜单 = 业务场景；账务管理 = 基于账务的数据管理容器。
- **单一真相源**：前端菜单配置集中在 `menuConfig.ts`；占位页统一 `Placeholder` 组件。
- **分层清晰**：后端 `api`（路由）/ `schemas`（校验）/ `models`（ORM）三层分离。

## 2. 技术架构

| 层 | 技术 |
|---|---|
| 前端 | Vue 3 + vue-router（hash 模式）+ Element Plus + axios + ECharts |
| 后端 | FastAPI + Pydantic + SQLAlchemy 2.0 |
| 数据库 | 开发 SQLite；生产 PostgreSQL（预留切换） |
| 构建 | 前端 Vite（`npm run build`，产物 `dist/` 被 gitignore） |

## 3. 模块划分（一级菜单）

| 一级 | 性质 | 子模块 |
|---|---|---|
| 首页 | 入口 | 仪表盘 |
| 合同管理 | 新增 | 人事/销售/采购合同 + 往来单位 |
| 税务管理 | 新增 | 税务工作台/个税/印花税/发票税务汇总 |
| 报销管理 | 新增 | 申请/审批/台账 |
| 工资管理 | 现有归并 | 工资/工资设置 |
| 资产管理 | 现有归并 | 固定资产 |
| 账务管理 | 现有归并（容器） | 总账/发票/出纳/报表/结账 |
| 设置 | 工具 | 基础数据/基础设置 |

## 4. 目录结构

```
frontend/src/
  layouts/        menuConfig.ts（菜单单一真相源）, MenuPanel.vue, MainLayout.vue
  router/         index.ts（路由）
  api/            dashboard.ts（axios 实例，待抽为共享 http）
  views/          dashboard/ general-ledger/ invoice/ settings/ ...（按模块分目录）
  types/          各模块 TS 类型
backend/app/
  main.py         FastAPI 入口，注册 router（prefix=/api）
  api/            <module>.py（路由）
  schemas/        <module>.py（Pydantic）
  db/             engine / session / base（待建）
  models/         ORM 模型（待建）
```

## 5. 关键技术点
- 菜单渲染：`MenuPanel.vue` 读取 `menuConfig.ts` 渲染子菜单。
- 占位机制：未开发页用 `Placeholder` 组件（`el-empty`）。
- 鉴权：axios 请求拦截器自动附加 `Authorization: Bearer <token>`（登录接口待建）。
