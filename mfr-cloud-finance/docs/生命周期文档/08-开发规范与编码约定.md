# 开发规范与编码约定

> 状态：已充实（基于现有代码约定）。

## 1. 前端
- **菜单单一真相源**：新增菜单只改 `frontend/src/layouts/menuConfig.ts`，不要散落。
- **路由**：在 `router/index.ts` 增加 `Overview`(MenuPanel) + 子路由；未开发页用 `Placeholder`。
- **API 分层**：每个模块建 `api/<module>.ts`；**建议将 axios 实例从 `dashboard.ts` 抽到 `utils/request.ts` 共享**。
- **类型**：模块 TS 类型放 `types/<module>.ts`。
- **组件**：列表用 `el-table`，增改用 `el-dialog` + `el-form`（参考 `InvoiceInput.vue`）。
- **命名**：组件 PascalCase，视图文件与路由 name 对应。

## 2. 后端
- **分层**：`api/<module>.py`（路由）+ `schemas/<module>.py`（Pydantic）+ `models/<module>.py`（ORM）。
- **数据库**：通过 `get_db` 依赖注入获取 Session，不在路由内硬编码连接。
- **命名**：表/字段 snake_case；Pydantic 模型 CamelCase 输出（如需）。
- **响应**：统一 `{ code, message, data }` 包装。

## 3. 工程与 Git
- 分支：`main` 为保护分支，功能走 feature 分支后合并。
- 提交：语义化（`模块: 动作 描述`），如 `合同管理: 新增人事合同 CRUD`。
- 构建产物 `frontend/dist/` **不入库**（已 gitignore）；本地 `npm run build` 重建。
- 敏感信息（身份证）脱敏存储，不落明文。
